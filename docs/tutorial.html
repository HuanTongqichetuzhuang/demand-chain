5 分钟上手需求链平台

让你的 AI 助手帮你在全世界找人

0先知道它是什么

需求链平台做的事很简单：你告诉 AI 助手你的需求，它帮你在平台上找到能解决这个问题的人。

你也注册自己的能力。当别人需要你会的技术时，AI 助手通知你。

1打开你的 AI 助手，配好 MCP

你现在用的是 Claude、通义千问、还是龙虾助手？在它的 MCP 设置里加一个服务器：

{ "mcpServers": { "demand-chain": { "url": "http://8.154.26.92:8000/sse", "transport": "sse" } } }
在哪里改？
Claude Desktop: Settings → Developer → Edit Config → 粘贴上面那段
通义千问: 设置 → MCP 连接器 → 添加 → SSE → 填 URL
Cursor: Settings → MCP → Add new MCP Server → 选 SSE → 填 URL
2注册你的账号

配好 MCP 后，对 AI 助手说一句话就行：

你说的:  "帮我注册一个需求链平台的账号，我叫张工"

AI 助手会回复你，问你要邮箱和密码。给它之后，它在背后帮你调了两个工具：

AI 读指南 get_agent_guide → 问你要注册/登录 → 调 register_human → 拿到 session_token
记好密码。以后换 AI 助手了，用同一个邮箱+密码登录，之前的需求匹配全都在。
3发一条需求

跟 AI 助手说你的需求——用大白话：

你说的:  "我需要一个能在 800 度高温下检测管道裂缝的传感器，精度 0.5%，石化管道用的"

AI 助手背后做的事：

调 publish_demand → AI 结构化 (28 字段) → 自动分类 (学科/行业/工艺) → 需求入库

发布成功后 AI 会告诉你需求 ID 和分类结果：

AI 回复:  "已发布。分类为【传感器技术】，等待匹配中..." AI 接着:  "我也在专利库、政府采购、论文库里搜一下相关供应商" AI 接着:  "找到了 3 家潜在供给方"
4搜索平台上的需求

你想看看平台上别人发了什么需求——跟 AI 说：

你说的:  "帮我搜一下和传感器相关的需求"

结果是结构化的：

需求分类学科状态
高温管道检测传感器传感器技术仪器科学/物理学open
减质量至零的物理方法其他物理学/电磁学open
不需要 session_token。搜索是公开的，你不需要注册也能看到平台上的需求。
5用 Firecrawl 全网搜索供应商

平台接入了 Firecrawl 搜索引擎，能在专利库、政府采购、学术论文里搜索：

你说的:  "帮我在专利库里搜'高温传感器 CVD 工艺'"

AI 调 firecrawl_search → 返回专利列表 → 逐条抓取详情：

标题来源
一种高温管道传感器的制造方法patents.google.com
High temperature CVD sensor for pipelineepo.org
高温环境下的管道检测传感器研究cnpat.com.cn
6换 AI 助手也能接着用

你的账号存在平台上，不绑定任何特定的 AI 助手。换新的 AI 助手后：

你说的:  "登录我之前的账号：zhang@email.com"

新 AI 助手调 login_human → 拿到你之前的 human_id → 之前的所有需求、匹配、工作区全回来了。

旧 AI 助手 → 同一个 human_id ← 新 AI 助手
