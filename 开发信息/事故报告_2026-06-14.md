1. **不同时段的Agent并行操作** — 我（原Agent）在6月12-13日做了大量修复和优化，另一个Agent在6月14日以"全新继承者"身份接管

### 恢复步骤

#### 第一步：VNC登录服务器（必须由你做）

1. 登录阿里云控制台 → https://ecs.console.aliyun.com
2. 找到 ECS 实例（8.154.26.92）
3. 点击"远程连接"→ VNC（或 Workbench）
4. 输入服务器密码进入终端

#### 第二步：检查Docker状态

VNC终端里执行：
```bash
# 看docker是否在运行
systemctl status docker

# 如果没运行就启动
systemctl restart docker

# 看容器是否存活
docker ps -a
```

#### 第三步：重新启动服务

```bash
cd /opt/demand-chain

# 确认docker-compose文件在
ls docker-compose.prod.yml

# 重启所有容器
docker compose -f docker-compose.prod.yml -p dc up -d

# 等10秒，看看是否跑起来了
docker ps
```

#### 第四步：检查服务是否恢复

```bash
# 检查Web
curl -s http://localhost:8080/ | head -3

# 检查MCP
curl -s http://localhost:8000/ | head -3

# 检查数据库数据
docker exec dc-db psql -U dc -d demand_chain -c "SELECT COUNT(*) FROM capability_profiles;"
docker exec dc-db psql -U dc -d demand_chain -c "SELECT COUNT(*) FROM demands;"
```

#### 第五步：告诉我结果

VNC进去了之后，把以下信息复制给我：
1. `docker ps` 的输出
2. `systemctl status docker` 的输出
3. 数据库 counts

然后我可以通过SSH接管后续修复。

### ⚠️ 如果容器启动失败（常见问题）

**现象**: 8080端口被占用、镜像名错误、配置文件缺失

解决方法（我可以远程帮忙，你先连上VNC就行）：
1. `docker logs dc-web` — 看Web容器错误日志
2. `docker logs dc-mcp` — 看MCP容器错误日志
3. 如果镜像不存在：`docker images | grep demand` 看有哪些镜像
4. 用正确的镜像名重启
